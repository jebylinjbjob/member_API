main User story 作為一個專案開發者，我希望統一發送通知給用戶

本專案會提供三種服務 SMTP SMS firebase

# 用戶故事

作為一個專案開發者，我希望統一發送通知給用戶，無論是透過 Email、簡訊或推播服務，讓我能夠更有效地與用戶溝通。

作為一個工程師，我希望能夠輕鬆的發送通知，並且可以使用API來溝通，集中管理不同的通知服務，提升開發效率

作為一個專案經理，我希望可以讓我追蹤專案的發送狀態以及使用路，可以方便我估算成本。

# 平台特性
這邊的使用者等於一組可以寄送的個體
和要寄給誰是不一樣的

所以我可以藉由發送出去的API_Key 去讓這使用者對所有想要發送的目標發送信件以及訊息

# 租戶
可以擁有多個使用者
以及所有使用者只要上傳 stmp sms  firebase 環境變數就可以共用這些環境變數
並且使用者只能自己的環境變數，但是可以讀取其他人設定的環境變數
